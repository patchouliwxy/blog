# 前端搜索与 Mock 数据的实战组合

当项目还没有后端时，`mock JSON + fetch` 是一种很高效的开发方式。

## 为什么先用 Mock API

它可以让我们先把列表、详情、搜索、标签筛选这些核心体验搭起来，再逐步替换成真实服务。

## 搜索实现思路

前端模糊搜索并不复杂，关键是把可以被检索的字段提前准备好。

### 推荐字段

- 标题
- 摘要
- 标签
- 正文关键词摘要

```ts
const matches = `${post.title} ${post.excerpt} ${post.searchableText}`
  .toLowerCase()
  .includes(keyword);
```

## 可扩展方向

后续如果文章规模扩大，可以考虑：

1. 构建时生成静态搜索索引
2. 使用更强的前端搜索库
3. 接入后端全文检索服务

## 小结

在博客项目早期，重要的是先把内容体验跑通，而不是过早依赖复杂基础设施。
